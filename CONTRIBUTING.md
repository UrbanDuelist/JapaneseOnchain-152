## 🤝 Code of Conduct

By participating in this project, you agree to keep our community welcoming and respectful. Be kind, be constructive, and help each other grow. We celebrate Japanese culture's values of respect (敬意) and harmony (和).

## 🐛 Reporting Bugs

Before reporting a bug, please:

1. **Check existing issues** — someone may have already reported it: [Issues](../../issues)
2. **Check Discussions** — it may be a known limitation: [Discussions](../../discussions)

When reporting, use the **Bug Report template** and include:
- A clear title starting with `[BUG]`
- Steps to reproduce the bug
- What you expected vs what happened
- Screenshots if helpful
- Your browser and OS

👉 [Create a Bug Report](../../issues/new/choose)

## 💡 Suggesting Features

Have an idea that would make JapaneseOnChain better?

1. First, **check existing ideas** in [Discussions → Ideas](../../discussions/categories/ideas)
2. If it's a small idea, post it in [Discussions → Ideas](../../discussions/categories/ideas)
3. If it's a larger feature, [open a Feature Request issue](../../issues/new/choose)

When suggesting, include:
- A clear title starting with `[FEATURE]`
- The problem it solves
- Your proposed solution
- Any mockups or examples

## 🚀 Your First Contribution

Not sure where to start? Look for issues tagged:

- `good first issue` — simple, well-scoped tasks great for newcomers
- `help wanted` — tasks where we would love community help
- `documentation` — improve or add docs (no coding required!)

**General workflow:**

1. Fork the repository on GitHub
2. Clone your fork locally
3. Create a new branch: `feature/your-feature-name` or `fix/your-bug-fix`
4. Make your changes
5. Commit with a clear message (see format below)
6. Push to your fork
7. Open a Pull Request on GitHub

---

## 📝 Commit Message Format

Use clear, descriptive commit messages with an emoji prefix:

| Emoji | Type | Example |
|---|---|---|
| ✨ | New feature | `✨ Add: Japanese NFT gallery page` |
| 🐛 | Bug fix | `🐛 Fix: broken link on mobile` |
| 📝 | Documentation | `📝 Update: README with setup guide` |
| 🎨 | Style/UI | `🎨 Improve: hero section layout` |
| ♻️ | Refactor | `♻️ Refactor: simplify API calls` |
| 🚀 | Performance | `🚀 Perf: optimize image loading` |
| 🔒 | Security | `🔒 Fix: sanitize user inputs` |

## 🔄 Pull Request Process

1. **Ensure your branch is up to date** with the main branch before submitting
2. **Describe your changes** clearly in the PR description
3. **Link any related issues** using `Closes #123` in the PR description
4. **Be responsive** — the maintainer may request changes or ask questions
5. Once approved, your PR will be merged — congratulations, you're a contributor! 🎉

### PR Checklist

- [ ] My changes work correctly on the live site
- [ ] I have tested in multiple browsers (Chrome, Firefox, Safari)
- [ ] I have updated the README if needed
- [ ] My commit messages follow the format above
- [ ] I have linked any related issues
